hexo搭建个人博客教程

  • 笔记 教程
  • 2017-11-27

一、gitHub的使用

1、本地创建ssh key
     桌面右键git bash here/windows+R，然后输入 ssh-keygen -t rsa -C "邮箱"
     记住ssh key的保存路径，然后一路回车，直到显示很多小圆圈
     默认路径：c:/users/用户名/.ssh
·
2、打开c:/users/用户名/.ssh文件夹下面的id_rsa.pub文件（打开方式选择记事本/editplus）
     ctrl+a 全选内容，然后ctrl+c复制备用

3、打开github.com, 登录个人账号，点击头像右侧小三角在下拉菜单里面选择settings
     在新页面左侧选择ssh and gpg keys，然后点击右侧绿色new ssh key按钮 
     title：随意填
     key : ctrl+v粘贴刚才复制的key，然后点击下方绿色add ssh key按钮

4、回到git命令行工具，输入ssh -T git@github.com 回车
     如果提示 Hi XXX! You've successfully authenticated, but GitHub does not provide shell access. 代表添加key成功

5、验证github邮箱和用户名：
     在git命令行窗口输入：
     git comfig --global user.email "邮箱" 回车 （没有任何提示代表验证成功）
     git comfig --global user.name "用户名" 回车
     注意：编写命令的时候注意英文符号以及必要的空格

6、github创建仓库
          github页面点击头像左侧加号，点击下拉列表new repository
     填写仓库名称，repository name，尽量保证是英文
     点击下方create repository按钮

7、桌面右键git bash here，然后执行/copy如下命令回车

     echo "# test" >> README.md|在桌面创建内容为”测试“的md文件
     git init                                               初始化git
     git add README.md                                      添加刚才生产的md文件
     git commit -m "first commit"                           编写更新说明
     git remote add origin git@github.com/用户名/test.git    设置需要提交的仓库
      git push -u origin master                              向仓库推送文件    
    注意：最后一步首次推送需要验证用户名密码，窗口会自动弹出
            第一次推送成功后，后续还需要推送文件怎么办？
             git add SECONDFILE.md (文件名.后缀)
            git commit -m "second commit" （更新说明）
            git push -u origin master

二、代码托管到github

1、在博客项目根目录的_config.yml文件里面添加如下信息：
     deploy:
          type: git
          repository: https://github.com/用户名/用户名.github.io
          branchy: master
     注意：格式的缩进跟_config.yml里面其他项保持一致

2、添加git依赖包
     命令行执行: npm install hexo-deployer-git --save

3、执行：hexo g  #编译博客项目

4、执行：hexo d  #将代码托管到github
     如果成功，可以通过用户名.github.io访问个人博客

三、域名解析

记录类型 主机类型 解析线路（isp） 记录值
CNAME www 默认 用户名.github.io
A @ 默认 192.30.252.153
A @ 默认 192.30.252.154

四、域名绑定到个人博客

1、在public文件下添加CNAME文件（不带后缀名）

2、在CNAME里面填入更让人域名（不带www）
